Parker used the AI SDK's streamText function with tool-based agentic retrieval, where the LLM decides when and what to retrieve from the knowledge base.

Upstash Vector: Used as the hybrid vector database with built-in embedding generation (bge-small-en-v1.5 for dense search and BM25 for keyword search), eliminating the need for a separate embedding API.

Vector Databases & RAG: Parker engineered a complete RAG system that connects a user's query to a knowledge base stored in a vector database. This demonstrates a strong, practical understanding of modern AI application architecture.

OR-Tools CP-SAT: Parker engineered a constraint-optimization solver microservice using Google's OR-Tools CP-SAT library in Python for the Sous project. The solver optimally assigns staff to station-time slots under hard and soft constraints including hour limits, clopening prevention, manager coverage, and fairness balancing.
